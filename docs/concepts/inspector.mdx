The **MetaMCP Inspector** is a powerful debugging tool that allows you to test, explore, and troubleshoot MCP servers and MetaMCP endpoints in real-time. Unlike the official MCP inspector, MetaMCP's inspector automatically saves server configurations and provides enhanced features for debugging complex setups.

## What is the Inspector?

The Inspector provides:

- **Real-time testing** of MCP servers and endpoints
- **Saved configurations** for quick access and debugging
- **Tool exploration** with interactive parameter testing
- **Request/response monitoring** with detailed logging
- **Error diagnosis** with helpful troubleshooting information
- **Performance metrics** for optimization insights

<Card title="Inspector Features" icon="magnifying-glass">
Similar to the official MCP inspector, but with **saved server configs** - MetaMCP automatically creates configurations so you can debug MetaMCP endpoints immediately without manual setup.
</Card>

## Accessing the Inspector

### Web Interface

1. **Navigate** to the MetaMCP dashboard
2. **Click** "MCP Inspector" in the sidebar
3. **Select** a server or endpoint to inspect
4. **Start testing** tools and exploring capabilities

### Direct URL Access

```bash
# Access inspector directly
http://localhost:12008/inspector

# Inspect specific endpoint
http://localhost:12008/inspector?endpoint=my-endpoint

# Inspect specific server
http://localhost:12008/inspector?server=hackernews
```

## Testing Scenarios

### Development Workflow

<AccordionGroup>
  <Accordion icon="code" title="New Server Testing">
    **When adding a new MCP server:**
    
    1. Add server configuration in MetaMCP
    2. Open inspector and select the server
    3. Verify connectivity and tool discovery
    4. Test individual tools with sample data
    5. Check for errors or performance issues
    6. Add to namespace once validated
  </Accordion>

  <Accordion icon="filter" title="Middleware Debugging">
    **When configuring middleware:**
    
    1. Create namespace without middleware
    2. Test baseline functionality
    3. Add middleware one by one
    4. Verify each middleware effect
    5. Check performance impact
    6. Optimize configuration based on results
  </Accordion>

  <Accordion icon="globe" title="Endpoint Validation">
    **Before deploying endpoints:**
    
    1. Test endpoint connectivity
    2. Verify authentication works
    3. Check all transport types (SSE, HTTP, OpenAPI)
    4. Test with actual client configurations
    5. Validate rate limiting and CORS
    6. Monitor performance under load
  </Accordion>
</AccordionGroup>

## Advanced Features

### Custom Scripts

Run custom test scripts against your endpoints:

```javascript
// Example test script
async function testWorkflow() {
  const tools = await inspector.listTools();
  
  for (const tool of tools) {
    try {
      const result = await inspector.callTool(tool.name, {
        // test parameters
      });
      console.log(`‚úÖ ${tool.name}: Success`);
    } catch (error) {
      console.log(`‚ùå ${tool.name}: ${error.message}`);
    }
  }
}
```
